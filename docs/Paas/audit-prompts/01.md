## What to look for

- **IDOR / broken object-level authorization**: a regular user passing
  `?userId=<victim>`, `?patientId=<victim>`, `?alertId=<victim>` and the
  handler using that value in the WHERE clause without verifying
  ownership. We already found one IDOR in `clinical-data` GET — look
  for the same pattern across `provider/**` especially.
- **Provider ↔ patient assignment gating**: any provider route that
  returns patient data must verify a `providerPatient` row exists,
  status active, providerId = session user.
- **Admin role checks**: admin routes using `withAdminAuth` vs just
  `withAuth`. Flag any admin surface gated only by `withAuth`.
- **Input validation**: every POST/PUT/PATCH/DELETE body must validate
  via Zod. Flag missing validation.
- **Rate limiting**: AI, auth, and cron endpoints must be rate-limited.
  Missing rate limits on signup/signin = account enumeration + brute
  force.
- **Account enumeration**: signup/signin should return identical
  responses whether the email exists or not.
- **Bearer token compares**: any admin/cron endpoint using `===` on a
  secret instead of `timingSafeEqual` (B-029/B-036 class).
- **Env-var-missing fallback to known-string placeholder**: any secret
  field that falls back to a literal when unset (B-029/Polar class).
- **Module-load-time env reads** that cap at a bad default (B-020/B-021
  lazy-init pattern).
- **CSRF**: non-GET routes must go through `withAuth` (which carries the
  CSRF origin check) or have their own equivalent.
- **PHI in logs**: `logger.info/warn/error` that includes free-text from
  user input, clinical data, distress text, etc. DWA has
  `CompliancePolicyZ.redactPromptsInAudit` but nothing enforces it.
- **Runtime `CREATE TABLE IF NOT EXISTS`**: schema drift vs the
  migration system. Already seen in `academy/feedback/route.ts`.
- **SSRF**: any `fetch()` built from a user-supplied URL.
- **`!==` against a Drizzle column object** — B-001 class, silently
  false.
- **Silent JSONB dot-path writes** (B-039): any
  `profileService.updateProfile(userId, { 'foo.bar': ... })` where
  `foo` might be null in the DB.

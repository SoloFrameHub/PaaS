## What to look for

- **Any file importing `pg`, `drizzle-orm/node-postgres`, or
  `@platform/tenancy/internal` directly** — record. The ESLint rule
  now blocks the internal subpath from apps, but the pre-existing
  `lib/db/index.ts` predates the rule.
- **Zod schema gaps**: `z.any()`, `z.record(z.any())`,
  `.passthrough()`, or missing `.strict()` on validation inputs.
- **SSRF-capable `fetch()`**: any fetch where the URL is built from
  user input, config, or upstream-response data.
- **Hardcoded secrets / tokens / URLs**: grep for
  `sk-`, `re_`, `Bearer `, `api_key=`, `password=` inside the files.
- **`dangerouslySetInnerHTML` in non-component files** (email HTML
  builders, PDF generators, RSS feeds).
- **PHI in `logger.*` calls**: any `logger.info/warn/error` with
  `{ text, content, message, body }` carrying raw user input is a
  HIPAA risk. Audit every call site.
- **Retry logic that re-uses the same DB transaction** after an error
  — causes Postgres "current transaction is aborted".
- **Rate-limit implementations** (`security.ts` + `redis.ts`): verify
  the Redis key is namespaced, fail-open vs fail-closed on Redis
  outage, no per-user cross-tenant collisions.
- **JSONB `sanitize-jsonb.ts`**: verify prototype pollution guards
  (`__proto__`, `constructor.prototype`, `prototype`) AND depth/size
  DoS.
- **Repositories**: must enforce userId ownership on every read/write.
  `postgresProfileRepository` — check for IDOR patterns (userId from
  caller vs from params).
- **ai/openai-coaching + ai/openai-flows + ai/rag**: prompt injection
  boundaries. User input should not be concatenated into the system
  prompt. RAG: vector query must be userId-scoped.
- **maia-client**: URL building SSRF-safe, timeouts set, text never
  logged or persisted beyond the classifier call.
- **storage/s3.ts**: bucket names / paths not user-controlled (path
  traversal into other users' files).
- **validations/**: schemas must match the handlers' actual
  expectations. Mismatch = validation bypass.
- **openapi/**: registry exposes only intended routes.
- **checkDistress**: safety-critical. If it fails, the whole crisis
  detection pipeline silently defaults to "none". Audit the fallback.

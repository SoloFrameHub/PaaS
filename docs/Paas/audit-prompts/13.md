## What to look for

### scripts/

- `content-pipeline.ts`, `content-quality-check.ts`,
  `docker-entrypoint.js`, `extract-quizzes.js`,
  `validate-curriculum.ts`, `index-lesson-embeddings`,
  `index-book-search`, `db-migrate`, and siblings.
- Any script that writes to prod by default.
- SQL built from string concatenation.
- Embedded Bearer / API keys.
- `require()` imports that bypass the ESM tree.

### seed-data/

- Bulk grep for `sk-`, `re_`, `@gmail|@yahoo|@hotmail|@outlook`
  (real-looking emails), phone numbers.
- Flag any seed entry that writes to a table that also serves real
  users in prod (risk: seed in prod spams user rows).

### e2e/

- Playwright `baseURL` — prod vs local?
- Hardcoded test credentials?
- Tests that mock a session cookie — only run when
  `NEXT_PUBLIC_MOCK_AUTH=true` AND `NODE_ENV !== 'production'`.

### apps/forms/ sub-app

- Ships as a **separate container** — audit end-to-end like a separate
  app.
- `next.config.js`: `output: 'standalone'`? `outputFileTracingRoot`?
- `entrypoint.js`: signal handling? runs as root?
- `app/api/forms/submit`: anti-spam, rate-limit, auth?
- `app/forms/[slug]`: path-traversal? SSRF on form config lookup?
- `lib/forms` + `lib/validations`: Zod strict?

### proxy.ts (renamed from middleware.ts in Phase 1)

- Re-verify the tenant-header strip is correct (B-030 class).
- Matcher must not exclude `/api/*`.

### next.config.js

- CSP, security headers.

### drizzle.config.ts

- Connection-string handling.

## What to look for

- **`dangerouslySetInnerHTML`**: every instance. Flag whether the HTML
  comes from user input, from content, or from a sanitized source.
  DWA has mental-health content; untrusted HTML from user free-text is
  an XSS path.
- **Uncontrolled `target="_blank"`** without `rel="noopener noreferrer"`.
- **Client-side secret exposure**: any `process.env.NEXT_PUBLIC_*` that
  references a value that should NOT be public (database URL, admin
  secret, webhook secret, AI API key). These get baked into the
  JavaScript bundle.
- **Client-side role/auth decisions**: any component that trusts a
  client-side flag (e.g., `isAdmin` from localStorage) for access
  control. Server-side checks are the only valid gate.
- **Hardcoded URLs pointing to internal services**.
- **Form submissions that skip CSRF**: any `fetch('/api/...', {method: 'POST'})`
  without proper handling. (Cookies are sent same-origin by default,
  but flag anything unusual.)
- **File upload components**: client-side validation is informational;
  server must enforce. Check for obvious bugs.
- **Uncontrolled iframe src / SSRF-in-browser**: `<iframe src={userData}>`.
- **Memory leaks**: `useEffect` without cleanup when a subscription,
  interval, or listener is registered.
- **`useSearchParams` / `useParams` trusted for auth decisions**.
- **`eval`, `new Function(...)`, `Function.constructor`** anywhere.
- **`document.cookie` reads / writes** (should be going through Lucia).
- **WebSocket / EventSource / SSE endpoints built from user input**.
- **React 19 `use()` with untrusted promises** (blocking SSR).
- **Types files**: no code risk but check for `any` leakage that would
  widen server/client contracts.
- **specs/**: spec/design docs. Read and record; flag if they describe
  behavior that's known-broken.

## What to look for

### Firebase residue

- Any file, import, or reference. If firebase is truly dead, the
  `lib/firebase/` directory is a landmine.
- Read every file in `lib/firebase/`; report whether it's still imported
  anywhere in the app.

### Integration SDKs (attio, hunter, notion, pipedrive, brevo, badgr, whatsapp, nodebb)

- Base URLs hard-coded vs env-configurable (prod/sandbox mismatch).
- Auth header built from env var — does it fall back to a placeholder
  when unset? (B-029 class.)
- Timeouts set on fetch calls.
- Retry strategy does not leak secrets into logs.
- Rate-limit / backoff on outbound calls.
- Error responses from upstream not exposed raw to users.

### whatsapp/ + email/

- Templated content: check for prompt / template injection where user
  input is interpolated into a template that renders HTML.

### firebase/ + db/

- Two DB clients in one app. Verify they don't both hold connections
  simultaneously (pool exhaustion).

### security.ts + redis.ts

- Same audit as DWA slice 02: rate-limit keys tenant-scoped, fail-open
  vs fail-closed on Redis outage.

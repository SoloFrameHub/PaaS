## What to look for

Same as prompt 03 (DWA components), plus GTM-specific:

- **Polar checkout buttons**: `NEXT_PUBLIC_POLAR_*` exposure. Server
  must verify the purchase, never trust client-declared amount or
  product.
- **Form components** that submit to `/api/forms/*`: client-side
  validation is fine but never treat client-validated data as
  trustworthy on the server.
- **Notion OAuth kickoff button**: `state` param must be server-
  generated and checked on callback.

# Prompt 06 — DWA _archive + Audits + docs-embedded (~790 files)

Audit the reference / archived / documentation directories inside DWA.
These are TS-excluded from the build but exist in the git tree and
could (a) contain leaked secrets, (b) contain old auth bypasses that
would be a landmine if re-imported, (c) be served statically by
mistake.

Repo: `/Volumes/ext-data/github/solofame-platform`.

Run `docs/audit-prompts/00.md` first if you haven't already. Follow
`CLAUDE.md` audit discipline.

## Scope

- `apps/dwa/_archive/` (~67 files)
- `apps/dwa/Audits/` (4 files) — prior audit artifacts
- `apps/dwa/_Full-Platform-Capabilities-Plans/` (1 file)
- `apps/dwa/docs/` (~677 files) — includes embedded Next.js template
  projects (`stellar-next/`, `mosaic-next/`)
- `apps/dwa/mental-health-refactoring-docs/` (14 files)
- `apps/dwa/load-tests/` (1 file)

## What to look for

- **Embedded secrets** (grep for `sk-`, `re_`, `Bearer `, `AKIA`,
  `PGPASSWORD=`, private SSH keys, `.env` fragments in MD files).
- **Old auth/admin code** that, if re-imported via autocomplete or a
  junior dev, would create a vulnerability. Flag specific files.
- **Static assets that could be served accidentally** if a default
  Next.js routing rule picks them up (check `public/**`, anything under
  `app/**` that looks like a page).
- **Research notes** containing internal URLs, employee emails,
  customer names (PII), or confidential business info that shouldn't
  live in a git tree at all.
- **`stellar-next`, `mosaic-next` template projects**: vendored
  templates. Treat as third-party; sweep for local modifications
  (files that clearly aren't template boilerplate).

## Procedure

1. Read all files under `Audits/`, `_Full-Platform-Capabilities-Plans/`,
   `mental-health-refactoring-docs/`, `load-tests/` line-by-line —
   these are small.
2. For `_archive/`: list every file. Categorize (archived route
   handler / archived lib / deleted script). Read every file
   end-to-end. Flag any with active secrets or obvious auth bypasses.
3. For `docs/`: bulk grep first (same patterns as Slice 05). Drill
   into any hits. Read all top-level `.md` files individually. For
   `docs/stellar-next/` and `docs/mosaic-next/` (vendored templates),
   grep for local mods vs template baseline.

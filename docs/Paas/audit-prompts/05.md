# Prompt 05 — DWA server/ (engine + data sweep, ~984 files)

Audit of `apps/dwa/server/**`. This directory contains the lesson
engine data (JSON), persona definitions, assessment instruments, and
some engine JS. Most files are content JSON; a few are executable.

Repo: `/Volumes/ext-data/github/solofame-platform`.

Run `docs/audit-prompts/00.md` first if you haven't already. Follow
`CLAUDE.md` audit discipline.

## Scope

Total files: ~984.

### Pass A — executable code (read line-by-line)

```bash
find apps/dwa/server -type f \( -name '*.ts' -o -name '*.tsx' -o -name '*.js' -o -name '*.mjs' -o -name '*.cjs' \) | sort
```

Expected count: ~20-50 files. Read every one end-to-end. Focus:

- engine code that loads JSON content at runtime
- `eval`, `new Function`, dynamic `require()` of strings
- JSONB sanitization seams
- `personas.js`, `curriculumData.js` — verify they're pure data, no
  side effects, no DB access

### Pass B — data files (pattern sweep + drill-down)

All JSON / MD / MDX / YAML under `apps/dwa/server/data/**` and any
other content subdirs.

Run these greps across the whole data tree; for every hit, open the
file and classify:

```bash
# Embedded secrets (API keys, bearer tokens, URL-embedded creds)
grep -rlnEi 'sk-[a-zA-Z0-9_-]{20,}|re_[a-zA-Z0-9_-]{20,}|AKIA[A-Z0-9]{16}|Bearer\s+[A-Za-z0-9_.-]{20,}' apps/dwa/server/data/ || true

# Internal URLs / dokploy / internal IPs / ssh
grep -rlnEi 'dokploy|ssh://|10\.|172\.(1[6-9]|2[0-9]|3[01])\.|192\.168\.' apps/dwa/server/data/ || true

# HTML/JS embedded in content that could bypass sanitization
grep -rlnEi '<script|javascript:|dangerouslySetInnerHTML|onerror=|onload=' apps/dwa/server/data/ || true

# Placeholder credentials forgotten in content
grep -rlnEi 'TODO.*(secret|key|password)|FIXME.*(secret|key|password)|password\s*=\s*["\x27][^"\x27]+["\x27]' apps/dwa/server/data/ || true

# Personal identifiers in sample data
grep -rlnEi '[a-zA-Z0-9._-]+@(gmail|yahoo|hotmail|outlook|aol|proton)\.' apps/dwa/server/data/ || true
```

Classify each hit: false positive / real embedded secret / placeholder.

### Pass C — sampled deep read

Read a random sample of ~20 files per top-level data subdir
(`assessments/`, `courses/`, `personas/`, `research/`, etc.) to verify
the content shape. Record the sample paths.

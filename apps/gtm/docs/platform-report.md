### State Persistence Architecture

```
User Interaction
    │
    ├──► localStorage (immediate, offline-safe)
    │

    On page load: Server state wins → falls back to localStorage
    On auth failure (401): Sets authFailed flag, stops server sync
```

## 3. Quiz & Assessment System

### Entrance Assessment (Onboarding)

A multi-step founder assessment runs during onboarding and shapes the entire learning journey.

**What it captures:**
- **DISC personality profile** — 10-question assessment producing a 2-letter type (e.g., DI, SC)
- **Founder category** — Self-identified type (solopreneur, growth-stage, coach, creator, service provider)
- **Business context** — Industry, stage, target roles, target customer type, product description, deal size

**What it produces (6 domain scores, 0–100):**
- `overallReadiness` — Composite readiness score
- `icpClarity` — How well-defined their Ideal Customer Profile is
- `positioningStrength` — Clarity of market positioning
- `messagingConsistency` — Message alignment across channels
- `channelReadiness` — Marketing channel infrastructure maturity
- `salesProcessMaturity` — Sales workflow sophistication

**Outputs that drive personalization:**
- `recommendedPath` — inbound, outbound, or hybrid acquisition strategy
- `recommendedStartCourse` — Where to begin the curriculum
- `quickWins[]` — Immediately addressable gaps
- `criticalGaps[]` — High-priority deficiencies
- `journeyMap[]` — Multi-phase progression plan
- `personalizedInsight` — AI-generated narrative summary

### In-Lesson Quizzes

Each lesson can contain embedded quizzes processed through the quiz service.

**Question types:**
- Multiple-choice (single correct answer)
- Single-choice (radio selection)
- True/false
- Reflection (open-ended, AI-evaluated)

**Scoring and feedback:**
- Automated scoring for objective questions
- AI-evaluated reflections (requires 50+ characters) with context from the user's founder profile
- Results persist to user profile and count toward XP/badges
- Reflection feedback is personalized — the AI knows the user's product, business model, deal size, and industry

### Roleplay Assessment

Full AI-powered roleplay sessions for practicing sales conversations.

**Structure:**
- User selects an industry and buyer role
- DISC type of the simulated buyer is set
- Multi-turn conversation transcript is captured
- AI evaluates the session with a structured rubric

**Evaluation output:**
- `score` — 0–100 numeric rating
- `strengths[]` — What the user did well
- `improvements[]` — Areas for growth
- `coachingMessage` — DISC-adapted coaching feedback

### Gamification Layer

| System | Description |
|--------|-------------|
| **XP** | Awarded per lesson completion; level-based progression with cumulative thresholds |
| **Streaks** | Consecutive-day tracking (timezone-aware); milestones at 3, 7, 14, 30 days |
| **Badges** | 24 badges across categories: Milestones, Streaks, XP, Artifacts, Roleplay, Community |
| **Activity Feed** | Tracks lesson completions, badge earns, course completions, artifact creation, streak milestones, community posts |

**Badge trigger events:** `lesson_completed`, `course_completed`, `track_completed`, `quiz_passed`, `roleplay_completed`, `roleplay_scored`, `artifact_saved`, `community_post`, `community_upvotes`, `streak_updated`

## 4. AI Integration

### AI Architecture

The platform uses a **dual-provider system**: OpenRouter (preferred, for model flexibility) with a direct OpenAI fallback. Two singleton clients handle different modalities:

- `aiClient` — Text generation, chat, structured JSON outputs
- `voiceClient` — Text-to-speech and speech-to-text

Model selection is task-specific via `resolveModel(task)`, and all usage is logged with `logTokenUsage(task, model, usage)`.

### AI-Powered Coaching Chat

The centerpiece AI feature — a contextual coaching assistant available on every lesson page.

**How it works:**
1. User asks a question via the `AILessonCoach` flyout component
2. The system builds a rich context payload including:
   - User's DISC type and learning style
   - Current course and lesson metadata
   - Assessment scores and recommended path
   - Previously saved artifacts (ICP docs, positioning statements)
   - Prior Q&A history for this lesson
3. The AI responds with coaching tailored to the user's personality type

**DISC-adapted response styles:**

| DISC Type | Coaching Style |
|-----------|---------------|
| **D (Dominant)** | Direct, results-focused, challenge-oriented. Gets to the point fast. |
| **I (Influencer)** | Enthusiastic, vision-focused, uses stories and analogies. |
| **S (Steady)** | Supportive, process-oriented, step-by-step guidance. |
| **C (Conscientious)** | Analytical, data-driven, logical reasoning with evidence. |

**Guardrails:**
- Rate-limited per user (`AI_RATE_LIMIT`)
- Configurable question limit per lesson (default: 3)
- Chat sessions persisted to Postgres for analytics

### AI Assessment & Evaluation

**Structured JSON outputs** — All AI evaluation flows use `response_format: { type: 'json_object' }` with explicit schema examples in the prompt, ensuring deterministic, parseable results.

**Assessment flow:**
- Onboarding questionnaire → AI analysis → 6-dimension scoring + recommended path
- 60-second timeout (vs. default 20s) for complex assessment processing
- Results stored as `assessmentSnapshot` for Metabase dashboards

**Quiz reflection evaluation:**
- AI grades open-ended reflections with full founder context
- Feedback personalized to user's product, business model, and deal size

**Roleplay evaluation:**
- Multi-turn transcript analysis
- DISC-aware scoring rubric
- Structured strengths/improvements/coaching output

### AI Forum Personas

Four AI characters participate in community forums, each with a distinct DISC personality:

| Persona | DISC Type | Role | Prompt File |
|---------|-----------|------|-------------|
| **Alex** | D (Dominant) | The Skeptic — challenges assumptions, asks hard questions | `lib/prompts/personas/alex.ts` |
| **Jordan** | I (Influencer) | The Builder — encourages, brainstorms, connects ideas | `lib/prompts/personas/jordan.ts` |
| **Morgan** | C (Conscientious) | The Perfectionist — focuses on details, quality, edge cases | `lib/prompts/personas/morgan.ts` |
| **Sam** | S (Steady) | The Mentor — provides steady guidance, emotional support | `lib/prompts/personas/sam.ts` |

**Behavioral intelligence:**
- **Topic-relevant selection** — Persona responds only when the thread topic matches their `topicPreferences` (0.7 relevance + 0.3 randomness weighting)
- **Natural cadence** — Only responds ~40% of the time for realism
- **4-hour cooldown** — Minimum gap between posts from the same persona
- **Thread ratio cap** — Bot posts capped at 20% of any thread
- **Fade-out logic** — Weekly evaluation of human vs. bot post ratios:
  - `humanRatio > 90%` → persona goes **silent** (community is self-sustaining)
  - `humanRatio > 80%` → persona enters **fading** (reduced response rate)
  - Otherwise → **active**

**Adaptive persona allocation per pod size:**
- 1 human member → 3 AI personas assigned
- 2 human members → 2 AI personas
- 3+ human members → 1 AI persona (facilitator only)

### AI Facilitator

An automated AI facilitator runs a weekly rhythm for each learning pod via Trigger.dev cron jobs.

**Weekly schedule:**
| Day | Post Type | Purpose |
|-----|-----------|---------|
| Monday | **Kickoff** | Theme introduction tied to pod's curriculum stage |
| Wednesday | **Nudge** | Mid-week engagement prompt, progress check-in |
| Friday | **Synthesis** | Week summary, member highlights, next-week preview |

**Context awareness:**
- Pulls per-member data: curriculum progress, post count this week, engagement level (high/moderate/quiet)
- Stage-specific themes: 5 rotating themes per curriculum stage (foundation, lead_gen, sales_conv)
- Stores weekly `facilitatorContext` snapshot: member profiles, pod health score, AI summary

**Guardrails:**
- Max 10 bot posts per pod per day
- Pod health score drives tone (struggling pods get more encouragement)

### AI Voice Features

- **Text-to-speech** — Audio streaming via OpenAI TTS for lesson content
- **Speech-to-text** — Transcription for voice-based interactions

### ICP Validation with Multi-Persona Feedback

When a user submits their ICP document, the AI runs it through four simulated buyer perspectives (mapped to DISC types), returning structured feedback from each angle — the skeptic, the enthusiast, the analyst, and the pragmatist.

## 6. Adaptability & Personalization

### Assessment-Driven Course Unlocking

The platform uses a **7-rule cascading unlock system** that adapts the available curriculum based on each user's assessment results and progress:

| Rule | Logic |
|------|-------|
| 1 | Course 0 is always unlocked |
| 2 | Sequential within tracks — complete Course N to unlock N+1 |
| 3 | Assessment-driven — unlocks `recommendedStartCourse` + quick-win and critical-gap courses |
| 4 | Cross-track unlocks — completing Course 3 opens path-specific tracks (inbound/outbound/hybrid) |
| 5 | Journey map phases — multi-phase progression based on assessment output |
| 6 | Track-first unlock — any unlock in a track opens the track's first course |
| 7 | Current course is always accessible |

This means no two learners see the same course sequence — a founder with strong positioning but weak prospecting infrastructure will be routed differently than one with the opposite profile.

### DISC-Personalized AI Coaching

Assessment data flows through **every layer** of the platform:

```
Onboarding Assessment
    │
    ├──► Course Unlock Sequence (which courses appear and in what order)
    ├──► AI Coaching Style (DISC-adapted response tone and structure)
    ├──► Pod Matching (DISC diversity + business context weighting)
    ├──► Persona Selection (fill DISC gaps in pod composition)
    ├──► Facilitator Rhythm (curriculum-stage-specific themes)
    ├──► Roleplay Evaluation (DISC-aware coaching feedback)
    ├──► Personalized Examples (dynamic in-lesson content)
    └──► Quiz Reflection Feedback (business-context-aware AI grading)
```

### Personalized In-Lesson Examples

The `PersonalizedExample` component dynamically adapts generic examples to the user's specific context:

- Template: `"If you are selling [terminology] to [buyer role]..."`
- Resolver pulls from the user's onboarding questionnaire data (product, industry, target audience)
- Visual indicator: **"Context-Aware"** (green) when personalized, **"Setup Required"** (amber) when context is missing
- Graceful fallback to generic examples if profile is incomplete

### Adaptive AI Forum Personas

The persona system self-regulates based on community engagement:

- **Persona allocation scales down** as pods grow (3 personas for 1 human → 1 for 3+ humans)
- **DISC gap filling** — personas are chosen to complement the existing DISC distribution in a pod
- **Fade-out logic** — personas autonomously reduce activity as humans become more active:
  - Active → Fading → Silent (based on 2-week human/bot post ratio)
- **Topic relevance scoring** prevents off-topic persona responses

### Adaptive Facilitator Context

The facilitator builds a per-member engagement profile each week:

| Engagement Level | Criteria |
|-----------------|----------|
| **High** | 3+ posts this week |
| **Moderate** | 1–2 posts this week |
| **Quiet** | 0 posts this week |

The facilitator adjusts its tone and prompts based on the pod's collective engagement — quiet pods receive more direct encouragement, while active pods get synthesis and challenge posts.

### Progress-Based Gamification

The streak, XP, and badge systems create a feedback loop that adapts to engagement patterns:

- **Streaks** are timezone-aware with a grace period (yesterday counts)
- **Badges** trigger on 10+ event types with race-condition protection
- **Milestone celebrations** fire on course/track completion
- **Activity feed** provides social proof of peer progress

### Accessibility

The platform implements core accessibility patterns across all interactive components:

- `aria-expanded`, `aria-controls`, `aria-labelledby` on accordions and expandable sections
- `aria-label` on interactive buttons (flip cards, exercises)
- `aria-checked` on checkboxes
- `aria-live="polite"` on live regions (template previews, AI responses)
- `aria-describedby` for hint references
- `aria-hidden="true"` on decorative icons
- Full keyboard navigation support (tab order, Enter/Space activation)

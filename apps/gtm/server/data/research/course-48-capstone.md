This course uses a different pattern than standard courses:

1. **System Assembly Phase** (Lessons 1-5) — Review, refine, and integrate all artifacts from Tracks 1-7 into a unified system. Each lesson focuses on a major system area. ~60 min each.

## LESSON 6: The 30-Day Sprint: Planning Your Targets (50 min)

### Key Topics

1. **Sprint Philosophy** — The 30-day sprint is not about doing everything. It's about executing the top priorities from your system assembly with discipline and measurement. Focus on 2-3 key outcomes, not 10.
2. **Selecting Sprint Priorities** — Based on Lessons 2-5 assembly, identify the 2-3 highest-leverage improvements. Use the ICE framework: Impact (1-10) x Confidence (1-10) x Ease (1-10). Score each possible sprint goal and pick the top 2-3.
3. **Setting SMART Sprint Targets** — Each sprint goal must be: Specific (send 200 outbound emails), Measurable (achieve 5% reply rate), Achievable (based on current capacity), Relevant (directly impacts revenue), Time-bound (within 30 days).
4. **Weekly Milestone Breakdown** — Week 1: Launch and calibrate. Week 2: Optimize based on early data. Week 3: Scale what's working. Week 4: Document results and plan next sprint. Each week has specific deliverables.
5. **Daily Activity Targets** — Based on the Weekly Rhythm from Course 21 (5-7 hours/week): Define exact daily activities and expected outputs. Example: Monday = 90 min prospecting (20 new leads), Tuesday = 60 min outreach (15 personalized emails), etc.
6. **Accountability System** — Daily: Log activity in sprint tracker. Weekly: Review metrics in guided session (Lessons 7-10). Sprint end: Compare results to targets. Share with peer or AI coach for feedback.

## LESSON 8: Week 2 Execution & Review (90 min)

### Key Topics

1. **Week 2 Execution Focus: Optimize and Iterate** — Week 1 data is in. Now optimize: A/B test subject lines, adjust personalization approach, try different call-to-action styles, test new LinkedIn message variants.
2. **A/B Testing Review** — If running email A/B tests: What won? By how much? Is the sample size meaningful (need 50+ per variant minimum for reliable results)? Implement winning variants.
3. **Conversion Funnel Analysis** — Trace the full funnel: Prospects contacted → Replies received → Conversations booked → Conversations held → Proposals sent → Deals closed. Where is the biggest drop-off?
4. **Pipeline Building Check** — By Week 2, you should have deals in pipeline. If not, the problem is upstream (not enough outreach, wrong targeting, or poor messaging). If deals are in pipeline but not moving, the problem is downstream (discovery, proposals, or follow-up).
5. **Mid-Sprint Assessment** — Are sprint targets achievable? If significantly behind, either: (a) the targets were unrealistic — adjust down, or (b) execution was insufficient — adjust effort up. Do not change goals; change approach.
6. **Week 3 Preview** — Week 3 is "scale what works." Set targets that double down on winning channels, messages, and approaches.

- Conversion funnel metrics (each stage)

## LESSON 9: Week 3 Execution & Review (90 min)

### Key Topics

1. **Week 3 Execution Focus: Scale What's Working** — Double down on the channels, messages, and approaches that produced results in Weeks 1-2. Cut anything that isn't performing. This is the "push" week.
2. **Volume Scaling** — If a sequence is getting 8% reply rates, can you send it to more prospects? If LinkedIn outreach is booking calls, can you increase connection requests? Scale up proven winners.
3. **Quality vs Quantity Balance** — Scaling does not mean spamming. Maintain personalization quality. The Draft + Human Gate model from Course 21 is critical here. More volume with the same quality = better results. More volume with lower quality = worse results.
4. **Closing Velocity Check** — Are deals closing within your target timeline? If deals are stalling at proposal or negotiation stage, review Course 19 closing techniques. Apply urgency without pressure.
5. **Retention Actions** — During the sprint, don't neglect existing customers. Check health scores, send a check-in email, ask for a testimonial if appropriate. Retention is part of the system.
6. **Week 4 Preview** — Week 4 is documentation and reflection. Start thinking about: What would you tell someone replicating your system? What would you do differently?

### What Gets Reviewed

- Week 3 activity log (scaled activities)
- Conversion rates (maintained quality while scaling?)
- Pipeline velocity (deals moving through stages?)
- Close/revenue metrics (any closed deals or commitments?)
- Retention check (existing customer health)
- Evidence of scaling decisions (what was doubled down on, what was cut)

2. **Sprint Results Summary** — For each sprint goal: What was the target? What was achieved? What percentage of target? What's the primary reason for over/under-performance?
3. **30-Day Sprint Scorecard** — Aggregate all metrics: total outreach sent, total replies, total conversations, total proposals, total closed deals, total revenue generated or in pipeline. Compare to pre-sprint baseline.
4. **ROI Analysis** — Time invested (hours) × hourly rate = time cost. Tool costs for the month. Revenue generated or pipeline created. What's the return? Is this sustainable?
5. **Lessons Learned** — Top 3 things that worked, top 3 things that didn't, top 3 surprises, the single biggest insight from the sprint. These become input for the 90-day roadmap.
6. **Transition to Sustainable Rhythm** — The sprint is a burst. Now transition to the 5-7 hour/week rhythm from Course 21. Which sprint activities become weekly habits? Which were one-time setup? What's the steady-state system?

## LESSON 11: Final System Documentation (60 min)

### Key Topics

1. **Why Documentation Matters** — Your system has no value if it lives only in your head. Documentation enables: hiring (hand off to a sales hire from Course 45), scaling (replicate the system across new markets), reliability (if you're sick for a week, the system runs without you), accountability (you can audit what's working and what isn't).
2. **The System Architecture Document** — One page (or two) that shows the complete system: Acquisition → Conversion → Retention → Advocacy loop, with all tools, automations, agents, and human touchpoints mapped.
3. **Process Documentation (SOPs)** — For each repeatable process: what triggers it, what are the steps, what tools are used, what's the expected output, what are the common failure modes. Use the SOP format from Course 43.
4. **Metrics Dashboard Documentation** — Where are metrics tracked, what's the review cadence, what are the thresholds for action, who is responsible (you, or future hire).
5. **AI Agent Documentation** — For each AI agent: what's its purpose, what's its input, what's its output, what are its constraints, how is it supervised, what are its failure modes. Use the agent spec from Course 27.
6. **The Living Document Principle** — This documentation is never "done." Set a calendar reminder to review and update quarterly. Date every version. Track changes over time.

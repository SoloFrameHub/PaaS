## STANDARD DUAL EXAMPLE PATTERNS

### Pattern 1: Inline Dual Examples

**Format:**
```
"[Concept]. Whether you're [B2B scenario] or [Creator scenario], [universal principle]."
```

**Example:**
```
"Discovery uncovers the gap between current state and desired state. Whether you're qualifying a marketing director for $5K software or an entrepreneur for $2K coaching, you're learning what's broken, what they've tried, and why now matters."
```

---

### Pattern 2: Side-by-Side Tables

**Use for:** Questions, processes, checklists

**Format:**
```markdown
| Concept | B2B Example | Creator Example |
|---------|-------------|-----------------|
| **X** | B2B version | Creator version |
| **Y** | B2B version | Creator version |
```

**Example:**
```markdown
| Discovery Question | B2B Application | Creator Application |
|-------------------|-----------------|---------------------|
| **Current situation** | "How are you managing leads today?" | "How are you getting clients now?" |
| **Pain point** | "What's broken about your current CRM?" | "What's frustrating about your current process?" |
```

---

### Pattern 3: Sequential Dual Examples

**Use for:** Case studies, scenarios, walkthroughs

**Format:**
```
**B2B Example:**
[Complete B2B scenario]

**Creator Example:**
[Complete creator scenario with parallel structure]
```

## PER-LESSON WORKFLOW (10-12 MIN)

### Step 1: Find/Replace (2 min)
- [ ] Run primary find/replace commands
- [ ] Review context-dependent replacements
- [ ] Check for any remaining "B2B buyer" or "B2B sales"

### Step 2: Identify Example Locations (1 min)
- [ ] Scan lesson for major concepts
- [ ] Mark 1-2 places that need dual examples
- [ ] Prioritize: Introduction, main teaching sections, practice exercises

### Step 3: Add Dual Examples (3-5 min)
- [ ] Use Pattern 1 for inline examples
- [ ] Use Pattern 2 for questions/processes
- [ ] Use Pattern 3 for case studies
- [ ] Copy/paste from Creator Example Library as needed

### Step 4: Update Introduction (2 min)
- [ ] Add "whether you sell to..." framing if missing
- [ ] Ensure hook is inclusive (not B2B-only)
- [ ] Verify lesson preview mentions both contexts

### Step 5: Review Exercises (2 min)
- [ ] Check practice exercises work for both B2B and creator
- [ ] Update prompts to be inclusive
- [ ] Add creator-specific example if needed

### Step 6: Test Rendering (1 min)
- [ ] Preview lesson in platform
- [ ] Check formatting (no broken Markdown)
- [ ] Verify links work
- [ ] Confirm exercises load correctly

**Total:** 10-12 minutes per lesson

## WHEN YOU GET STUCK

**Problem:** Not sure if terminology change is needed
**Solution:** Ask "Would a creator founder understand this?" If no, change it.

**Problem:** Can't think of creator example
**Solution:** Use Creator Example Library above, or use parallel B2B → Creator mapping:
- Software → Program/Course
- CRM → Kajabi/Teachable
- Marketing Director → Course Creator
- $5K deal → $2K program

**Problem:** Example feels forced or token
**Solution:** Skip it. Only add examples that strengthen teaching.

**Problem:** Running behind schedule
**Solution:** Focus on high-priority lessons (1, 2, 13 for each course) + bulk find/replace for others

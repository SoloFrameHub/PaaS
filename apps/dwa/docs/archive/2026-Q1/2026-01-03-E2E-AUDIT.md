## 🔴 Critical Feature Failures (Research Required)
Based on the latest full-suite run (41 tests), significant regressions are present in downstream features.

### 1. Tools & Community (`tools-and-community.spec.ts`)
**Result**: 20+ Failures / 0 Passes
- **ICP Builder**: Failing on navigation and form display.
    - *Probable Cause*: Route changes (e.g., `/academy/tools/icp-builder` vs `/icp-builder`) or Next.js App Router structure mismatches.
- **Community**: 404 errors on `/community/feed` and `/community/forum`.
    - *Probable Cause*: The directory structure shows `app/(default)/community/feed/page.tsx`, but the test might be hitting the wrong URL or there's a middleware redirect loop.
- **Notifications**: Fails to open panel.

### 2. Courses (`courses.spec.ts`)
**Result**: Mixed (8 Passed, 6 Failed)
- **Passing**: Quiz functionality (Answer, Submit, Retake) and basic Catalog navigation.
- **Failing**:
    - `should open lesson content`: The test cannot find the lesson content element.
    - `should mark lessons as complete`: The "Mark as Complete" button selector is stale or the API call is failing.
    - `should show completed lessons in sidebar`: Logic mismatch in how progress is displayed vs how it's stored.

### 3. AI & Evaluation (`roleplay.spec.ts`)
**Result**: Active Blockers
- **Evaluation Flow**: Fails to show performance results.
    - *Probable Cause*: The mock evaluation data structure in the test doesn't match the new Mosaic UI result cards.

---

## 🛠️ Summary of Blockers
1.  **URL Mismatches**: The shift to Next.js App Router has moved many pages, but specs are still looking for Express-style URLs or old directory patterns.
2.  **Stale Selectors**: The new "Mosaic" design system uses different class names and button structures that the tests haven't been updated for.
3.  **Missing "data-testid"**: Many interactive elements Lack unique identifiers, making tests fragile to layout shifts.

## 📋 Recommended Research Areas (For USER)
1.  **Route Mapping**: Confirm if `/community` and `/icp-builder` are intended to be under `(default)` or elsewhere.
2.  **State Logic**: Investigate why "Mark as Complete" doesn't reflect in the sidebar immediately (Hydration or Context lag?).
3.  **Evaluation Card**: Compare the JSON returned by the evaluation flow with the frontend component props.

## 🔍 Root Cause: User ID Mismatch

### The Problem

When we create test fixtures, we use:
```typescript
userId: 'test-user-completed'
```

But when the mock authentication creates a session, it generates:
```typescript
userId: 'mock-testexamplecom'  // Based on email
```

### What Happens

1. Test calls `signInWithCompletedOnboarding(page)`
2. Helper creates profile with `userId: "test-user-completed"` ✅
3. Helper signs in with `email: "test@example.com"`
4. Mock auth creates session with `userId: "mock-testexamplecom"` ❌
5. App tries to load profile for `userId: "mock-testexamplecom"`
6. Profile not found → `profileService.getOrCreateProfile()` creates NEW profile
7. New profile has `onboardingCompleted: false` ❌
8. User redirected to `/onboarding/welcome` ❌

### Evidence

From `/tmp/soloframehub_mock_profiles.json`:
```json
{
  "mock-testexamplecom": {
    "userId": "mock-testexamplecom",
    "onboardingCompleted": false  // ← Auto-created profile
  },
  "test-user-completed": {
    "userId": "test-user-completed",
    "onboardingCompleted": true   // ← Our fixture (unused)
  }
}
```

---

## 🔧 Solution

We need to align the userId in our fixtures with the userId generated by mock auth.

### Option A: Match Mock Auth UserId Format

Update `e2e/test-data.ts` to use the same userId format as mock auth:

```typescript
export const completedOnboardingUser = {
    email: 'test@example.com',
    password: 'password123',
    userId: 'mock-testexamplecom',  // ← Match mock auth format
    profile: createBaseProfile('mock-testexamplecom', 'test@example.com', {
        // ...
    }),
};
```

### Option B: Update Mock Auth to Use Custom UserIds

Modify the mock auth session creation to accept a custom userId:

```typescript
// In /api/auth/session route
if (mockMode && customUserId) {
    session = { uid: customUserId, email };
} else {
    session = { uid: `mock-${email.replace(/[@.]/g, '')}`, email };
}
```

### Option C: Use Email-Based Lookup

Instead of userId, look up profiles by email in the mock repository:

```typescript
// In helpers.ts
export async function signInWithCompletedOnboarding(page: Page) {
    const user = testUsers.completed;
    
    // Save profile by email instead of userId
    await page.evaluate(async (userData) => {
        await fetch('/api/test/setup-profile-by-email', {
            method: 'POST',
            body: JSON.stringify({ email: userData.email, profile: userData.profile }),
        });
    }, user);
    
    await signIn(page, user.email, user.password);
}
```

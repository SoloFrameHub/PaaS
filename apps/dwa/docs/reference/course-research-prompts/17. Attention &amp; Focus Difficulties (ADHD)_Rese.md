# 17. Attention \& Focus Difficulties (ADHD)

Research Prompt:
Review adult ADHD presentations, including inattention, hyperactivity–impulsivity, rejection sensitivity, emotional regulation issues, and common comorbidities. Summarize evidence-based non-pharmacological supports (CBT for ADHD, skills training, organizational systems, time management techniques, environmental modifications) and map them to Next.js tools like checklists, planners, and timers (e.g., components/adhd/TaskBreakdownTool.tsx, components/adhd/PomodoroTimer.tsx). Design anonymous self-reflection modules around executive function challenges using TS types such as ExecutiveSkill, ExecutiveSkillRating, with on-device feedback that normalizes difficulties and suggests strategies rather than diagnosing ADHD. Include UX guidance tailored for ADHD (minimal visual clutter, clear calls to action, short steps per page, offline-friendly local state when possible) and a11y practices for keyboard navigation and screen readers.

Adult ADHD often involves **inattention**, hyperactivity–impulsivity, emotional dysregulation (including rejection sensitivity), and frequent comorbidities, and you can translate core non‑pharmacological supports into short, anonymous Next.js tools without diagnosing ADHD. Below is a concise, implementation‑oriented blueprint.[^1][^2][^3][^4][^5][^6]

***

## Clinical framing for adult ADHD

- Core symptoms in adults: persistent inattention (disorganization, forgetfulness, difficulty sustaining focus), and varying levels of hyperactivity–impulsivity (restlessness, interrupting, acting without thinking).[^2][^5][^1]
- Emotional dysregulation is common: intense, rapidly shifting emotions, irritability, and difficulty calming down; many adults report strong sensitivity to criticism or perceived rejection (“rejection sensitivity”).[^7][^8][^3][^4]
- Common comorbidities: anxiety disorders, depression, substance use disorders, and other neurodevelopmental conditions, all of which add to functional impairment.[^9][^3][^5][^1]

Your course should:

- Normalize these patterns as understandable brain‑based differences.
- Emphasize that tools are skills training and not a substitute for assessment or medication decisions.

***

## Evidence‑based non‑pharmacological supports → Next.js tools

Evidence supports CBT for adult ADHD, skills training, and environmental modifications that target time management, organization, planning, and emotional regulation.[^10][^11][^12][^6]

Key skill domains and how to map them:

1. Task breakdown and planning (`TaskBreakdownTool.tsx`)
    - CBT/skills element: break large tasks into small, concrete steps; estimate time; reduce overwhelm.[^11][^12][^6]
    - Tool behavior: simple wizard that takes a “big task” and guides user to 3–7 steps with time estimates and a “first tiny step.”
2. Time management and focus (`PomodoroTimer.tsx`)
    - CBT/skills element: structured work/break cycles, external timers, clear start/stop to reduce procrastination and time‑blindness.[^12][^6][^11]
    - Tool behavior: adjustable work/break intervals, local state only, optional checklist of “distractions to park.”
3. Organizational systems (checklists, routines)
    - CBT/skills element: external reminders, consistent routines, visual checklists for daily anchors (morning, work start, shutdown routine).[^6][^10][^12]
    - Tool behavior: reusable checklist templates stored in `localStorage`, with “starred” items for essentials.
4. Environmental modifications
    - CBT/skills element: reduce visual/auditory distractions, prepare workspace, use cues (sticky notes, alarms) to support focus.[^10][^11]
    - Tool behavior: brief interactive “environment audit” checklist with suggestions; no PHI.

## Example: `TaskBreakdownTool.tsx`

### Types

```ts
// components/adhd/types.ts
export interface TaskStep {
  id: string;
  description: string;
  estimatedMinutes?: number;
  done: boolean;
}

export interface TaskPlan {
  id: string;
  title: string;
  context?: string; // e.g. "home", "work", "study"
  whyItMatters?: string;
  steps: TaskStep[];
}
```


### Component

```tsx
// components/adhd/TaskBreakdownTool.tsx
"use client";

import { FormEvent, useState } from "react";
import { TaskPlan, TaskStep } from "./types";
import { v4 as uuidv4 } from "uuid";

export function TaskBreakdownTool() {
  const [step, setStep] = useState<1 | 2>(1);
  const [plan, setPlan] = useState<TaskPlan>({
    id: uuidv4(),
    title: "",
    context: "",
    whyItMatters: "",
    steps: [],
  });
  const [newStep, setNewStep] = useState<{ description: string; estimatedMinutes: string }>({
    description: "",
    estimatedMinutes: "",
  });
  const [errors, setErrors] = useState<string[]>([]);

  const validatePlan = () => {
    const e: string[] = [];
    if (!plan.title.trim()) e.push("Give your task a short title.");
    if (plan.steps.length === 0)
      e.push("Add at least one small step.");
    setErrors(e);
    return e.length === 0;
  };

  const handleAddStep = () => {
    if (!newStep.description.trim()) return;
    const step: TaskStep = {
      id: uuidv4(),
      description: newStep.description,
      estimatedMinutes: newStep.estimatedMinutes
        ? Number(newStep.estimatedMinutes)
        : undefined,
      done: false,
    };
    setPlan((p) => ({ ...p, steps: [...p.steps, step] }));
    setNewStep({ description: "", estimatedMinutes: "" });
  };

  const handlePlanSubmit = (ev: FormEvent) => {
    ev.preventDefault();
    if (!validatePlan()) return;
    setStep(2);
  };

  const toggleStepDone = (id: string) => {
    setPlan((p) => ({
      ...p,
      steps: p.steps.map((s) =>
        s.id === id ? { ...s, done: !s.done } : s
      ),
    }));
  };

  return (
    <section aria-labelledby="task-tool-heading">
      <h2 id="task-tool-heading">Break a Task into Small Steps</h2>
      <p>
        This tool helps you turn a big task into clear, doable steps. It is
        educational and anonymous; avoid entering names or other identifying
        details.
      </p>

      {errors.length > 0 && (
        <div
          role="alert"
          aria-live="assertive"
          className="error-summary"
        >
          <h3>Check these fields</h3>
          <ul>
            {errors.map((e) => (
              <li key={e}>{e}</li>
            ))}
          </ul>
        </div>
      )}

      {step === 1 && (
        <form onSubmit={handlePlanSubmit} noValidate>
          <div>
            <label htmlFor="title">Big task</label>
            <input
              id="title"
              name="title"
              type="text"
              value={plan.title}
              onChange={(e) =>
                setPlan((p) => ({ ...p, title: e.target.value }))
              }
            />
          </div>

          <div>
            <label htmlFor="why">
              Why does this matter to you? (optional)
            </label>
            <textarea
              id="why"
              name="why"
              rows={2}
              value={plan.whyItMatters}
              onChange={(e) =>
                setPlan((p) => ({ ...p, whyItMatters: e.target.value }))
              }
            />
          </div>

          <fieldset>
            <legend>Small steps</legend>
            <div>
              <label htmlFor="step-desc">Step description</label>
              <input
                id="step-desc"
                name="step-desc"
                type="text"
                value={newStep.description}
                onChange={(e) =>
                  setNewStep((s) => ({ ...s, description: e.target.value }))
                }
              />
            </div>
            <div>
              <label htmlFor="step-time">
                Estimated time (minutes, optional)
              </label>
              <input
                id="step-time"
                name="step-time"
                type="number"
                min={1}
                value={newStep.estimatedMinutes}
                onChange={(e) =>
                  setNewStep((s) => ({
                    ...s,
                    estimatedMinutes: e.target.value,
                  }))
                }
              />
            </div>
            <button type="button" onClick={handleAddStep}>
              Add step
            </button>

            <ul aria-label="Planned steps">
              {plan.steps.map((s) => (
                <li key={s.id}>
                  {s.description}
                  {s.estimatedMinutes
                    ? ` (${s.estimatedMinutes} min)`
                    : null}
                </li>
              ))}
            </ul>
          </fieldset>

          <button type="submit">Use checklist</button>
        </form>
      )}

      {step === 2 && (
        <div>
          <h3>Checklist</h3>
          <p>
            Try starting with just the first tiny step. It is okay if you do not
            finish everything at once.
          </p>
          <ul>
            {plan.steps.map((s) => (
              <li key={s.id}>
                <label>
                  <input
                    type="checkbox"
                    checked={s.done}
                    onChange={() => toggleStepDone(s.id)}
                  />
                  {s.description}
                </label>
              </li>
            ))}
          </ul>
        </div>
      )}
    </section>
  );
}
```

## Example: `PomodoroTimer.tsx` (simple, ADHD‑friendly)

```tsx
// components/adhd/PomodoroTimer.tsx
"use client";

import { useEffect, useState } from "react";

type TimerMode = "work" | "break";

export function PomodoroTimer() {
  const [mode, setMode] = useState<TimerMode>("work");
  const [secondsLeft, setSecondsLeft] = useState(25 * 60);
  const [isRunning, setIsRunning] = useState(false);

  useEffect(() => {
    if (!isRunning) return;
    const id = window.setInterval(() => {
      setSecondsLeft((s) => {
        if (s <= 1) {
          const nextMode = mode === "work" ? "break" : "work";
          setMode(nextMode);
          return nextMode === "work" ? 25 * 60 : 5 * 60;
        }
        return s - 1;
      });
    }, 1000);
    return () => window.clearInterval(id);
  }, [isRunning, mode]);

  const minutes = Math.floor(secondsLeft / 60)
    .toString()
    .padStart(2, "0");
  const seconds = (secondsLeft % 60).toString().padStart(2, "0");

  const label =
    mode === "work"
      ? "Focus block"
      : "Short break";

  return (
    <section aria-labelledby="timer-heading">
      <h2 id="timer-heading">Simple Focus Timer</h2>
      <p>
        Use short focus blocks with planned breaks. This timer runs only in your
        browser and does not store any personal data.
      </p>

      <div
        role="status"
        aria-live="polite"
        className="timer-display"
      >
        <p>{label}</p>
        <p aria-label={`${minutes} minutes ${seconds} seconds`}>
          {minutes}:{seconds}
        </p>
      </div>

      <div>
        <button type="button" onClick={() => setIsRunning(true)}>
          Start
        </button>
        <button type="button" onClick={() => setIsRunning(false)}>
          Pause
        </button>
        <button
          type="button"
          onClick={() => {
            setIsRunning(false);
            setMode("work");
            setSecondsLeft(25 * 60);
          }}
        >
          Reset
        </button>
      </div>
    </section>
  );
}
```

You can later add simple presets (15/25/45 minutes) with radio buttons and labels.

## Anonymous executive‑function self‑reflection

Research links adult ADHD to executive difficulties in planning, time management, working memory, and emotion regulation. You can create a non‑diagnostic reflection module that helps users notice which skills are hard and suggests strategies.[^8][^3][^4][^5][^1]

### Types

```ts
// components/adhd/executiveTypes.ts
export type ExecutiveSkill =
  | "attention"
  | "planning"
  | "organization"
  | "timeManagement"
  | "workingMemory"
  | "emotionalRegulation"
  | "impulseControl";

export type ExecutiveSkillRating = 0 | 1 | 2 | 3 | 4; // 0=easiest, 4=very hard

export interface ExecutiveSkillItem {
  id: string;
  skill: ExecutiveSkill;
  prompt: string;
}

export interface ExecutiveReflection {
  ratings: Record<ExecutiveSkill, ExecutiveSkillRating>;
  notes?: string; // non-identifying
}
```


### Reflection component (on‑device, normalizing)

```tsx
// components/adhd/ExecutiveReflection.tsx
"use client";

import { useState } from "react";
import {
  ExecutiveSkill,
  ExecutiveSkillRating,
  ExecutiveSkillItem,
} from "./executiveTypes";

const ITEMS: ExecutiveSkillItem[] = [
  {
    id: "attn",
    skill: "attention",
    prompt: "Staying focused on one task until it is done",
  },
  {
    id: "time",
    skill: "timeManagement",
    prompt: "Starting tasks on time and finishing by deadlines",
  },
  {
    id: "org",
    skill: "organization",
    prompt: "Keeping track of things (e.g., papers, tools, tabs)",
  },
  {
    id: "plan",
    skill: "planning",
    prompt: "Breaking big tasks into smaller steps",
  },
  {
    id: "wm",
    skill: "workingMemory",
    prompt: "Remembering what you were doing after a distraction",
  },
  {
    id: "emo",
    skill: "emotionalRegulation",
    prompt: "Calming down when you feel frustrated or overwhelmed",
  },
];

const ratingLabels: Record<ExecutiveSkillRating, string> = {
  0: "Not a challenge",
  1: "A little hard",
  2: "Moderately hard",
  3: "Quite hard",
  4: "Very hard",
};

export function ExecutiveReflection() {
  const [ratings, setRatings] = useState<
    Partial<Record<ExecutiveSkill, ExecutiveSkillRating>>
  >({});
  const [notes, setNotes] = useState("");
  const [submitted, setSubmitted] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const handleChange = (
    skill: ExecutiveSkill,
    value: ExecutiveSkillRating
  ) => {
    setRatings((prev) => ({ ...prev, [skill]: value }));
  };

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    const skills = ITEMS.map((i) => i.skill);
    const missing = skills.filter((s) => ratings[s] === undefined);
    if (missing.length > 0) {
      setError("Please rate each area before viewing suggestions.");
      setSubmitted(false);
      return;
    }
    setError(null);
    setSubmitted(true);
  };

  const getSuggestion = (skill: ExecutiveSkill, rating: ExecutiveSkillRating) => {
    if (rating <= 1)
      return "This area feels more manageable right now.";
    switch (skill) {
      case "planning":
        return "Try using the task breakdown tool to turn big tasks into a few clear steps.";
      case "timeManagement":
        return "A timed focus block (like a simple timer with short breaks) may help you get started.";
      case "organization":
        return "Consider a small daily reset (5–10 minutes) to put key items back in the same place.";
      case "attention":
        return "Experiment with working on one task at a time while parking distractions on a separate list.";
      case "workingMemory":
        return "Try writing down the current step before switching tasks, so you can quickly pick it back up.";
      case "emotionalRegulation":
        return "Brief pause cues (three slow breaths, naming what you feel) can support your focus skills.";
      default:
        return "Noticing this challenge is already a useful step toward adapting your tools.";
    }
  };

  return (
    <section aria-labelledby="exec-heading">
      <h2 id="exec-heading">Executive Skills Self‑Reflection</h2>
      <p>
        This exercise helps you notice which skills feel easier or harder at the
        moment. It is anonymous and educational only; it does not diagnose ADHD
        or any other condition.
      </p>

      <form onSubmit={handleSubmit}>
        {ITEMS.map((item) => (
          <fieldset key={item.id}>
            <legend>{item.prompt}</legend>
            {(Object.keys(ratingLabels) as unknown as ExecutiveSkillRating[]).map(
              (value) => (
                <div key={value}>
                  <input
                    type="radio"
                    id={`${item.id}-${value}`}
                    name={item.id}
                    value={value}
                    onChange={() =>
                      handleChange(item.skill, value as ExecutiveSkillRating)
                    }
                  />
                  <label htmlFor={`${item.id}-${value}`}>
                    {ratingLabels[value as ExecutiveSkillRating]}
                  </label>
                </div>
              )
            )}
          </fieldset>
        ))}

        <div>
          <label htmlFor="notes">
            Notes for yourself (avoid names or identifying details)
          </label>
          <textarea
            id="notes"
            name="notes"
            rows={3}
            value={notes}
            onChange={(e) => setNotes(e.target.value)}
          />
        </div>

        {error && (
          <div role="alert" className="error-message">
            {error}
          </div>
        )}

        <button type="submit">View suggestions</button>
      </form>

      {submitted && (
        <section aria-live="polite">
          <h3>Your skill suggestions</h3>
          <p>
            Many people, including those with ADHD and other neurotypes, find
            these skills challenging. These ideas are starting points you can
            experiment with.
          </p>
          <ul>
            {ITEMS.map((item) => {
              const rating = ratings[item.skill] ?? 0;
              return (
                <li key={item.id}>
                  <strong>{item.prompt}</strong>:{" "}
                  {getSuggestion(item.skill, rating as ExecutiveSkillRating)}
                </li>
              );
            })}
          </ul>
        </section>
      )}
    </section>
  );
}
```

## UX and a11y guidelines tailored for ADHD

Drawing from neurodiversity‑aware UX guidance, ADHD‑friendly design benefits from minimal clutter, clear hierarchy, and chunked steps.[^13][^14][^15]

- Layout and navigation
    - Use clean, minimalist layouts with plenty of whitespace; avoid busy sidebars and multiple competing calls to action on one screen.[^14][^15][^13]
    - Keep navigation consistent across the ADHD course; key buttons (“Next,” “Start timer,” “Save steps”) should appear in the same location on each page.[^15][^14]
    - Break flows into short steps (1–3 inputs per screen) with a simple progress indicator (“Step 1 of 3”).[^14][^15]
- Content and interaction
    - Use brief, direct text and bullet points; avoid long dense paragraphs.
    - Make calls to action obvious (“Start 10‑minute focus,” “Add one step”) and avoid ambiguous labels.
    - Offer “good enough” usage guidance (e.g., “Using this once or twice a week is still useful”).
- Accessibility and keyboard / screen readers

```
- Semantic HTML (`<main>`, `<section>`, `<button>`, `<nav>`, `<fieldset>`, `<legend>`).  
```

    - Every input has a `<label htmlFor>` and any helper text is linked via `aria-describedby`.
    - Error messages use `role="alert"` or `aria-live="assertive"` and plain language.
    - Ensure full keyboard support with visible focus outlines and no keyboard traps.
    - Avoid auto‑playing media or flashing content; keep animations minimal and optional.[^13][^15]
- Offline‑friendly and anonymous
    - Store plans and reflections in `localStorage`/`IndexedDB`; do not collect names, emails, or third‑party identifiers.
    - Add concise privacy notes: “Stored only on your device. Please avoid entering names, addresses, or other identifying information.”
    - If you ever allow exporting text to share with clinicians, make it explicit that exporting/sharing is optional and happens outside the platform.

This setup gives you a clinically grounded, ADHD‑aware module that offers practical tools and self‑reflection while staying anonymous, non‑diagnostic, and accessible.[^1][^11][^12][^15][^6][^10][^13][^14]

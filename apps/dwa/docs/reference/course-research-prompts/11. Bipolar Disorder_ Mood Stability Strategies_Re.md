# 11. Bipolar Disorder: Mood Stability Strategies

Research Prompt (Next.js + TypeScript aware):
Investigate current evidence on bipolar I and II, cyclothymia, and mixed features, emphasizing psychoeducation, mood episode patterns, and relapse prevention. Identify best-practice mood tracking methods (sleep, activity, irritability, early warning signs) that can be safely implemented as anonymous, client-side self-monitoring tools in a Next.js app (e.g., components/bipolar/MoodTracker.tsx using TypeScript interfaces like MoodEntry). Review CBT for bipolar disorder, IPSRT (interpersonal and social rhythm therapy), and medication adherence psychoeducation to design modular lessons that map to React components (psychoeducation sections, quiz components, and scenario-based decision trees). Include guidance for building accessible forms (labels, error messaging, keyboard navigation) and ensuring that no PHI or identifiable data is stored without explicit, compliant backend design.

Bipolar mood stability can be supported with structured psychoeducation, client‑side mood tracking, and modular CBT/IPSRT‑informed tools that never store identifiable data by default. Below is a course-and-Next.js–oriented blueprint you can adapt directly.[^1][^2][^3]

## Clinical focus and psychoeducation

- Bipolar I: At least one full manic episode (often with major depression), higher risk of hospitalization and mixed features.[^4][^5][^1]
- Bipolar II: At least one hypomanic episode plus major depression, no history of full mania, often more depressive burden.[^5][^1][^4]
- Cyclothymia: Chronic (≥2 years) fluctuating subthreshold hypomanic and depressive symptoms, with relatively few symptom‑free periods.[^1][^4]
- Mixed features: Simultaneous or rapidly alternating manic/hypomanic and depressive symptoms, associated with higher suicide risk and treatment complexity.[^4]

Key psychoeducation themes for lessons:

- Chronic, relapsing course; need for long‑term mood and rhythm monitoring.[^2][^1][^4]
- Role of sleep, activity level, substances, and interpersonal stress in triggering episodes.[^3][^4]
- Importance of medication adherence and coordinated care with a prescriber.[^6][^7][^3]


## Evidence‑based relapse prevention and mood tracking

Psychoeducation and structured self‑monitoring reduce relapse and improve adherence, especially in group formats.[^8][^9][^7][^6]

Core variables to track (client‑side only):

- Mood: Daily rating on a simple scale (e.g., −3 to +3 or emoji faces).[^10][^2]
- Sleep: Bedtime, wake time, total hours, and perceived sleep quality, as early warning signals.[^11][^2]
- Activity/energy: Subjective energy, psychomotor agitation/retardation, and daily step targets.[^2][^10]
- Irritability and impulsivity: Brief 0–10 ratings, as these may rise before hypomania/mania.[^10]
- Medication taken (yes/no) without specifying drug names, if you want adherence‑oriented psychoeducation.[^3][^2]
- Custom early warning signs: e.g., racing thoughts, needing less sleep, overspending, social withdrawal.[^3]

Relapse‑prevention lesson content:

- Build an individualized early‑warning list and a written “If I notice X for Y days, I will do Z” plan (e.g., call clinician, adjust routine, reduce stimulants).[^6][^3]
- Teach limits on self‑management: the tool is for self‑awareness, not to change prescribed medications or replace care.


## Next.js + TypeScript mood tracking design (anonymous, client‑side)

You can implement a local‑only tracker using `localStorage` or `IndexedDB`, without any network calls.

Example types:

```ts
// components/bipolar/types.ts
export type MoodLevel = -3 | -2 | -1 | 0 | 1 | 2 | 3;

export interface MoodEntry {
  id: string; // UUID generated client-side
  dateISO: string; // e.g., "2026-02-09"
  mood: MoodLevel;
  sleepHours?: number;
  bedtime?: string;   // "23:30"
  wakeTime?: string;  // "07:00"
  irritability?: number; // 0–10
  energy?: number;      // 0–10
  impulsivity?: number; // 0–10
  medicationTaken?: boolean;
  earlyWarningFlags?: string[]; // user-defined labels only
  notes?: string; // free text; warn user not to enter names/addresses
}
```

Client‑side storage hook:

```ts
// hooks/useMoodEntries.ts
import { useEffect, useState } from "react";
import { MoodEntry } from "@/components/bipolar/types";

const STORAGE_KEY = "bipolar_mood_entries_v1";

export function useMoodEntries() {
  const [entries, setEntries] = useState<MoodEntry[]>([]);

  useEffect(() => {
    if (typeof window === "undefined") return;
    const raw = window.localStorage.getItem(STORAGE_KEY);
    if (!raw) return;
    try {
      setEntries(JSON.parse(raw));
    } catch {
      // ignore malformed data
    }
  }, []);

  useEffect(() => {
    if (typeof window === "undefined") return;
    window.localStorage.setItem(STORAGE_KEY, JSON.stringify(entries));
  }, [entries]);

  const addEntry = (entry: MoodEntry) => {
    setEntries((prev) => {
      const filtered = prev.filter((e) => e.dateISO !== entry.dateISO);
      return [...filtered, entry].sort((a, b) =>
        a.dateISO.localeCompare(b.dateISO)
      );
    });
  };

  return { entries, addEntry };
}
```

Client‑only component wrapper (Next.js 13+):

```tsx
// components/bipolar/MoodTracker.tsx
"use client";

import { FormEvent, useState } from "react";
import { useMoodEntries } from "@/hooks/useMoodEntries";
import { MoodEntry, MoodLevel } from "./types";
import { v4 as uuidv4 } from "uuid";

export function MoodTracker() {
  const { entries, addEntry } = useMoodEntries();
  const todayISO = new Date().toISOString().slice(0, 10);
  const [form, setForm] = useState({
    dateISO: todayISO,
    mood: 0 as MoodLevel,
    sleepHours: "",
    irritability: "",
  });

  const onSubmit = (e: FormEvent) => {
    e.preventDefault();
    const newEntry: MoodEntry = {
      id: uuidv4(),
      dateISO: form.dateISO,
      mood: Number(form.mood) as MoodLevel,
      sleepHours: form.sleepHours ? Number(form.sleepHours) : undefined,
      irritability: form.irritability
        ? Number(form.irritability)
        : undefined,
    };
    addEntry(newEntry);
  };

  return (
    <section aria-labelledby="mood-tracker-heading">
      <h2 id="mood-tracker-heading">Daily Mood & Sleep Check‑in</h2>
      <p>
        This tool is stored only on your device. Please do not enter names,
        addresses, or other identifying details.
      </p>
      <form onSubmit={onSubmit} noValidate>
        {/* Accessible fields go here (see next section) */}
      </form>
      {/* Simple history list or chart component */}
      <ul aria-label="Mood history">
        {entries.map((entry) => (
          <li key={entry.id}>
            {entry.dateISO}: mood {entry.mood}, sleep{" "}
            {entry.sleepHours ?? "n/a"} hours
          </li>
        ))}
      </ul>
    </section>
  );
}
```

To preserve anonymity:

- Do not link entries to accounts, emails, or IP in this module.
- Place a clear on‑screen statement that data stays in the browser unless the user chooses to sync with a clinician‑approved backend.
- If you later add sync, require explicit consent, strong auth, and a documented HIPAA‑aligned backend architecture (encrypted in transit and at rest, access controls, audit logs).


## Accessible, PHI‑safe forms

Form accessibility patterns:

```
- Use semantic `<form>`, `<fieldset>`, `<legend>`, `<label>` with `htmlFor` tied to `id`.  
```

- Provide helper text via `aria-describedby`.
- Expose errors using `aria-invalid="true"` and an error element with an `id` referenced by `aria-describedby`.
- Ensure full keyboard support (Tab order, visible focus ring, `role="alert"` for error summaries).
- Use high‑contrast color combinations and adequate font sizes; avoid color‑only distinctions.

Example form markup:

```tsx
// inside MoodTracker form
<fieldset>
  <legend>Today&apos;s check‑in</legend>

  <div>
    <label htmlFor="dateISO">Date</label>
    <input
      id="dateISO"
      name="dateISO"
      type="date"
      value={form.dateISO}
      onChange={(e) =>
        setForm((f) => ({ ...f, dateISO: e.target.value }))
      }
      required
    />
  </div>

  <div>
    <label htmlFor="mood">Mood (−3 = very low, 0 = neutral, 3 = very high)</label>
    <input
      id="mood"
      name="mood"
      type="number"
      min={-3}
      max={3}
      value={form.mood}
      onChange={(e) =>
        setForm((f) => ({ ...f, mood: Number(e.target.value) as MoodLevel }))
      }
      required
    />
  </div>

  <div>
    <label htmlFor="sleepHours">Hours of sleep (last night)</label>
    <input
      id="sleepHours"
      name="sleepHours"
      type="number"
      min={0}
      max={24}
      step="0.5"
      value={form.sleepHours}
      onChange={(e) =>
        setForm((f) => ({ ...f, sleepHours: e.target.value }))
      }
    />
  </div>
</fieldset>

<button type="submit">Save today&apos;s entry</button>
```

Privacy/HIPAA messaging for the UI:

- Banner text: “This tool is anonymous and runs in your browser only. Do not enter your name, contact details, addresses, or medical record numbers.”
- If you ever enable export (e.g., CSV download), warn users that downloaded files can become PHI if shared with a clinician and must be stored securely.


## CBT, IPSRT, and adherence mapped to React lessons

Evidence summary:

- CBT for bipolar focuses on identifying early warning signs, challenging extreme beliefs about mood states, and building structured routines; it improves depressive symptoms and time to relapse when combined with medication.[^12][^3]
- IPSRT stabilizes daily routines and interpersonal roles; it uses tools like the Social Rhythm Metric to track wake time, bedtime, mealtimes, and social contacts, improving symptom control and adherence.[^13][^3]
- Psychoeducation (often group‑based) improves knowledge, medication adherence, and reduces relapse and hospitalization risk.[^9][^7][^8][^6]

High‑level module structure:


| Module | React components | Clinical focus |
| :-- | :-- | :-- |
| 1. Understanding Bipolar | `<PsychoeducationSection />`, `<TimelineIllustration />` | Diagnosis, bipolar I/II/cyclothymia/mixed features, course, treatment overview.[^1][^4][^5] |
| 2. Mood \& Sleep Tracking | `<MoodTracker />`, `<SleepPatternTips />` | Mood charting, sleep hygiene, early warning signs, limits of self‑monitoring.[^11][^2][^10] |
| 3. CBT Skills | `<CBTConcepts />`, `<ThoughtRecordQuiz />` | Triggers, thoughts–feelings–behaviors model, coping plans, risky‑behavior management.[^12][^3] |
| 4. IPSRT \& Routines | `<SocialRhythmPlanner />`, `<RoutineStabilityQuiz />` | Regular wake/sleep, meals, activity, interpersonal stress management.[^13][^3] |
| 5. Medication \& Safety | `<AdherencePsychoeducation />`, `<AdherenceScenarioTree />` | How meds help, common barriers, adherence strategies, safety and crisis planning.[^6][^8][^3][^7] |

// components/bipolar/PsychoeducationSection.tsx
type SectionId = "bipolar-types" | "mixed-features" | "course-treatment";

interface SectionProps {
  section: SectionId;
}

const content: Record<SectionId, { title: string; paragraphs: string[] }> = {
  "bipolar-types": {

    paragraphs: [
      "Bipolar I includes at least one manic episode and often episodes of depression.",
      "Bipolar II includes hypomanic episodes and major depressive episodes, without full mania.",
      "Cyclothymia involves years of milder ups and downs that never reach full episode criteria."
    ]
  },
  // ...
};

export function PsychoeducationSection({ section }: SectionProps) {
  const { title, paragraphs } = content[section];
  return (
    <article aria-labelledby={`${section}-heading`}>
      <h2 id={`${section}-heading`}>{title}</h2>
      {paragraphs.map((p, idx) => (
        <p key={idx}>{p}</p>
      ))}
    </article>
  );
}

Quiz component (self‑assessment, no PHI):

```tsx
// components/bipolar/Quiz.tsx
"use client";

import { useState } from "react";

interface Question {
  id: string;
  prompt: string;
  options: { id: string; label: string; isCorrect: boolean }[];
}

const questions: Question[] = [
  {
    id: "q1",
    prompt:
      "Which change might be an early warning sign of mania or hypomania?",
    options: [
      { id: "a", label: "Needing much less sleep but still feeling very energetic", isCorrect: true },
      { id: "b", label: "Slightly tired after a late night once", isCorrect: false },
      { id: "c", label: "No change in energy or sleep", isCorrect: false }
    ]
  }
];

export function Quiz() {
  const [answers, setAnswers] = useState<Record<string, string>>({});
  const [submitted, setSubmitted] = useState(false);

  const onSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    setSubmitted(true);
  };

  return (
    <form onSubmit={onSubmit} aria-describedby="quiz-instructions">
      <p id="quiz-instructions">
        Select the best answer for each question. This quiz is anonymous.
      </p>
      {questions.map((q) => (
        <fieldset key={q.id}>
          <legend>{q.prompt}</legend>
          {q.options.map((opt) => (
            <div key={opt.id}>
              <input
                type="radio"
                id={`${q.id}-${opt.id}`}
                name={q.id}
                value={opt.id}
                onChange={() =>
                  setAnswers((prev) => ({ ...prev, [q.id]: opt.id }))
                }
              />
              <label htmlFor={`${q.id}-${opt.id}`}>{opt.label}</label>
            </div>
          ))}
          {submitted && answers[q.id] && (
            <p role="status">
              {q.options.find((o) => o.id === answers[q.id])?.isCorrect
                ? "Correct — this pattern often signals mood elevation."
                : "That is not the best answer. Review the lesson and try again."}
            </p>
          )}
        </fieldset>
      ))}
      <button type="submit">Check answers</button>
    </form>
  );
}
```

Scenario‑based decision tree for adherence:

- Present fictional vignettes (“Alex notices they are sleeping only 3 hours and stopped meds”), ask the learner to choose a response (“Contact prescriber”, “Increase dose alone”, “Ignore changes”), and give feedback consistent with guidelines (e.g., do not change doses without medical advice).
- Implement as nested questions in a single component; keep all data hard‑coded and generic, with no user‑specific text.


## PHI, anonymity, and backend guardrails

To keep this module HIPAA‑aligned by default:

- Default design: no network requests, no auth, no user identifiers; everything runs in the browser.
- Explicit warnings wherever free‑text notes exist: “Avoid including names, addresses, or other details that could identify you or other people.”
- If you later integrate log‑in or cloud sync:
    - Treat all stored mood data as PHI once linked to an identifiable account.
    - Use secure APIs behind authentication; encrypt in transit and at rest; restrict access by role; log access for audit.
    - Provide clear consent text explaining what is stored, where, and how it may be used.

This structure lets you build a bipolar mood‑stability course that is clinically grounded, interactive, and safely anonymous at the client level while remaining ready for future compliant backend extensions.[^7][^8][^6][^2][^10][^3]
